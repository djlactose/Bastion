# Pinned Python dependencies for the Bastion web app.
#
# These pins exist for supply-chain defense: without them, `docker build
# --no-cache` pulls whatever is "latest" at build time, exposing the image
# to typosquat / compromised-release attacks and making two consecutive
# builds non-reproducible.
#
# To regenerate with transitive-dep hash pinning:
#   pip install pip-tools
#   echo '<top-level package list from Dockerfile>' > web/requirements.in
#   pip-compile --generate-hashes --output-file web/requirements.txt web/requirements.in
# Then update the Dockerfile pip install to use `--require-hashes`.
